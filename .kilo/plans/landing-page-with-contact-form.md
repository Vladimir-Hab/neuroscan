# План: Превращение сайта в продающую страницу с формами и интеграцией Robokassa

## Ключевая особенность

**Важное требование:** Продукт требует индивидуальной настройки и установки, поэтому нельзя просто продать лицензию. **Имя и телефон клиента должны быть собраны ДО оплаты**, чтобы вы могли связаться с ними для настройки.

---

## Цель

Преобразовать информационный сайт в продающую страницу с:
1. **Две формы:**
   - ContactForm - для общих запросов и услуг
   - OrderForm - для покупки лицензии с обязательным сбором контактов
2. **Секция Pricing** с тарифами и кнопками "Купить"
3. **Интеграция с n8n** - прием заявок и работа с Robokassa
4. **Подготовка к Robokassa** - обработка платежей и callback

## Требования Robokassa (для проверки)
- ✅ Прямая ссылка без переадресации
- ✅ Все страницы под единым доменным именем
- ✅ Сайт корректно открывается, без тестовых данных
- ✅ Доступ должен быть открыт (или предоставлены данные авторизации)
- ✅ Сайт не должен быть в статусе "в разработке"

---

## Диаграмма потоков

### Flow 1: Заказ лицензии
```
Пользователь → Pricing → "Купить" 
→ OrderForm (modal) 
→ [валидация: имя, телефон] 
→ Отправка на n8n (type=order) 
→ n8n создает заказ + возвращает order_id + robokassa_url 
→ Редирект на Robokassa 
→ Оплата 
→ Robokassa callback → n8n (проверка подписи) 
→ n8n обновляет статус → Уведомление вам 
→ Редирект на payment-success
```

### Flow 2: Обратная связь
```
Пользователь → CTA / Услуги → "Оставить заявку"
→ ContactForm 
→ [валидация: имя, телефон] 
→ Отправка на n8n (type=contact) 
→ n8n сохраняет заявку → Уведомление вам 
→ Показать "Спасибо"
```

## Архитектура данных

### ContactForm → n8n параметры
```typescript
{
  type: 'contact',
  name: string,
  contact: string,  // телефон или telegram
  configuration: string,
  services: string[],  // массив услуг
  task: string
}
```

### OrderForm → n8n параметры
```typescript
{
  type: 'order',
  name: string,       // обязательно
  contact: string,    // обязательно (телефон)
  email: string,      // опционально
  configuration: string,
  tariff_id: string,  // ut-11-5, bp-3-0, full-kit
  tariff_name: string,
  price: number,
  comment: string     // опционально
}
```

### n8n → OrderForm ответ
```typescript
{
  success: true,
  order_id: number,
  robokassa_url: string  // полный URL с подписью
}
```

### Robokassa → n8n callback
```typescript
{
  InvId: number,        // order_id
  OutSum: string,       // сумма
  SignatureValue: string, // подпись
  Shp_Contact: string,  // телефон
  Shp_Name: string,     // имя
  Shp_Email: string,    // email
  Shp_Configuration: string
}
```

### 2.3 OrderForm - Форма заказа

**Назначение:** Для покупки лицензии с обязательным сбором контактных данных ДО оплаты

**КРИТИЧЕСКИ ВАЖНО:** Имя и телефон - обязательные поля, без которых заказ не может быть создан

**Логика работы:**
1. Пользователь нажимает "Купить" в карточке тарифа → открывается модальное окно с OrderForm
2. Пользователь заполняет форму (имя, телефон - СТРОГО обязательно)
3. Форма отправляется на n8n с параметром `type=order` и выбранным тарифом
4. n8n возвращает `order_id` → фронтенд формирует URL Robokassa → редирект
5. Robokassa отправляет callback на сайт (success/fail)
6. Callback отправляется на n8n для обновления статуса заявки

**Поля формы:**
1. **Имя** (обязательно) - для связи с клиентом перед настройкой
2. **Телефон** (обязательно) - для связи с клиентом перед настройкой
3. **Email** (опционально) - для отправки чека и документов
4. **Конфигурация 1С** (предзаполняется на основе тарифа, можно изменить):
   - 1С:Управление торговлей 11
   - 1С:Бухгалтерия предприятия 3.0
   - 1С:Комплексная автоматизация
   - 1С:ERP
   - 1С:Управление небольшой фирмой
   - Другая
5. **Комментарий** (опционально) - пожелания по настройке

**Компонент:** `src/components/OrderForm.tsx`
- Использовать Dialog из shadcn/ui (модальное окно)
- Параметры: `tariff` (название тарифа), `price` (цена в рублях), `tariffId` (идентификатор тарифа)
- Валидация: имя и телефон - СТРОГО обязательно, блокировать кнопку "Перейти к оплате"
- После успешной отправки на n8n → сформировать URL Robokassa → редирект

**Формирование URL Robokassa:**
```
https://auth.robokassa.ru/Merchant/Index.aspx?
  MerchantLogin=<login>&
  OutSum=<price>&
  InvId=<order_id>&
  Description=<название тарифа>&
  SignatureValue=<md5 хэш (login:price:order_id:password)>&
  Shp_Contact=<телефон>&
  Shp_Name=<имя>&
  Shp_Email=<email>&
  Shp_Configuration=<конфигурация>&
  Culture=ru
```

**Важно:** MD5 подпись должен формироваться на бэкенде (n8n), не на фронтенде

### 2.4 Обработка ответа n8n

**ContactForm:**
- 200 OK → Показать сообщение "Заявка отправлена. Мы свяжемся с вами в ближайшее время"
- Ошибка → Показать сообщение с описанием ошибки
- Отправлять данные как GET запрос с query параметрами
- Очищать форму после успешной отправки

**OrderForm:**
- 200 OK → n8n возвращает JSON с `order_id` и `robokassa_url`
  - n8n должен сформировать подпись и полный URL Robokassa
  - Фронтенд просто делает `window.location.href = response.robokassa_url`
- Ошибка → Показать ошибку "Не удалось создать заказ", не редиректить
- Показывать индикатор загрузки "Создание заказа..."

### 2.5 Обработка callback от Robokassa

**Создать страницы:**
- `/payment-success` - успешная оплата
- `/payment-fail` - неуспешная оплата

**Параметры от Robokassa (в URL):**
- `InvId` - номер заказа
- `OutSum` - сумма платежа
- `SignatureValue` - подпись
- `Shp_Contact`, `Shp_Name`, `Shp_Email`, `Shp_Configuration` - кастомные параметры

**Логика на страницах:**
1. Пользователь приходит на страницу после оплаты
2. Страница отправляет данные на n8n webhook для обновления статуса заявки
   - n8n проверяет подпись
   - n8n отправляет уведомление вам о новой покупке
3. Пользователь видит сообщение:
   - Success: "Оплата прошла успешно! Мы свяжемся с вами в ближайшее время для настройки"
   - Fail: "Оплата не прошла. Попробуйте снова или свяжитесь с нами"

## Этап 3: GitHub Pages деплой

### 3.1 Настроить GitHub Actions

**Создать файл:** `.github/workflows/deploy.yml`

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          VITE_BASE_URL: /${{ github.event.repository.name }}

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## Этап 4: Подготовка к Robokassa

### 4.1 Настройка n8n для работы с Robokassa

**Webhook:**
- Обновить существующий webhook для приема двух типов заявок:
  - `type=contact` - обратная связь
  - `type=order` - заказ на покупку

**Логика n8n для type=order:**
1. Принять данные формы (имя, телефон, email, конфигурация, тариф, цена)
2. Создать запись в вашей CRM/1С
3. Сгенерировать уникальный `order_id`
4. Сформировать подпись Robokassa на бэкенде:
   - MD5 хэш: `merchant_login + ":" + price + ":" + order_id + ":" + password_1`
5. Сформировать полный URL Robokassa
6. Вернуть JSON:
   ```json
   {
     "success": true,
     "order_id": 12345,
     "robokassa_url": "https://auth.robokassa.ru/Merchant/Index.aspx?..."
   }
   ```

**Логика n8n для Robokassa callback:**
1. Создать новый webhook: `/webhook/robokassa-callback`
2. Принять параметры (InvId, SignatureValue, OutSum)
3. Проверить подпись на бэкенде:
   - MD5 хэш: `out_sum + ":" + inv_id + ":" + password_2`
4. Обновить статус заказа в CRM/1С
5. Отправить уведомление вам (Telegram/Email)
6. Вернуть "OK" для Robokassa
